A. 🥇 품사 필터링 강화 (현재 코드 기반 개선)

현재 rag_pipeline.py의 simple_tokenize 함수는 검색에 유의미한 품사(예: 명사, 동사, 형용사)만 선택하여 토큰을 추출하고 있습니다.

    문제점: 일반적으로 조사(Josa), 어미(Eomi), 부호(Symbol) 등은 검색에 방해되므로 제거해야 합니다. 하지만 **고유 명사(NNP)**는 중요도가 높으므로 살려야 합니다.

    개선 방안: Kiwipiepy의 품사 태그를 사용하여 **검색에 꼭 필요한 품사(명사, 용언의 어간, 외국어 등)**만 선택하도록 필터링 조건을 강화합니다.

B. 🥈 불용어(Stopwords) 제거

검색의 정확도를 떨어뜨리는 단어(예: "오늘", "내일", "그", "이것", "저것", "그리고")를 토큰 리스트에서 제거하여 검색 노이즈를 줄입니다.

C. 🥉 특수 형태 처리 (복합 명사 처리)

"금융학부"와 같이 두 개 이상의 명사가 결합된 복합 명사를 하나의 토큰으로 유지하여 검색의 의미를 명확하게 합니다. (대부분의 현대 형태소 분석기는 이를 기본으로 처리합니다.)
